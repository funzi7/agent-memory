# paper-trader dashboard — cc-latest handoff

_Generated by a Claude Code session in the firewalled dev sandbox; bidi tokenizer
proven by unit cases + live-DOM inspection in a headless Chromium._

## What this change did
Fixed a bidi bug where `$10k` rendered with the `k` detached to the LEFT of the
number (`k$10`) in the RTL target/cash lines. Now `$10k` stays one intact,
left-to-right unit. Scope: `funzi7/agent-memory` only (`paper-trader/app.js`,
`paper-trader/sw.js`).

## Root cause
In `app.js`'s single bidi pass (`applyRtl` → `wrapRuns` → `TOKEN_RE`), the
currency rule `SIGNED_SRC` matched `$10` and **stopped at the `k`**
(`\$\d[\d.,]*%?` doesn't include a trailing letter). The leftover `k` was then
matched by the Latin-run rule as its own `<bdi>k</bdi>`. Two adjacent isolates in
an RTL line order right-to-left, so the `k` isolate landed to the LEFT of the `$10`
isolate — the reported `k$10`. The display strings (`על $10k`, the `$ לחודש (על
$10k)` toggle) were already single literals, so the split was purely in the
tokenizer, not the string construction.

## The fix (one regex, no wording change)
Prepend a currency-with-trailing-letter alternative to `SIGNED_SRC` so a `$`-amount
plus a letter suffix is captured as ONE LTR run:
```
SIGNED_SRC = "\\$\\d[\\d.,]*[A-Za-z][A-Za-z0-9]*%?|[+\\-]\\$?\\d[\\d.,]*%?|\\$\\d[\\d.,]*%?"
```
It must come first so `$10k`/`$10K`/`$100k`/`$1.5k` match whole; the plain
`$<digits>` alternatives still handle `$80.00`/`$100`/`+$5`. Each match goes into the
LTR group → `<bdi dir="ltr">$10k</bdi>` (sign/`$` on the left, `k` glued on the
right). `NONRTL_RUN` (the rulebook's Latin-run regex) is unchanged, so bare `10k`
still isolates as one run and the rulebook self-checks still pass.

## Verify (live + unit)
- Tokenizer unit cases: `$10k`/`$10K`/`$100k`/`$1.5k` → each a single LTR run;
  `10k` (no `$`) → one bare `<bdi>` run; `$80.00`/`$100`/`+$5`/`-12`/`+3.5%` →
  unchanged.
- Live DOM (headless Chromium): the `%`-mode preview renders
  `≈ <bdi dir="ltr">$0.80</bdi> לחודש על <bdi dir="ltr">$100</bdi> · <bdi dir="ltr">$80.00</bdi> לחודש על <bdi dir="ltr">$10k</bdi>`
  — `$10k` is ONE LTR isolate. The `$`-mode toggle label renders
  `$ לחודש (על <bdi dir="ltr">$10k</bdi>) ⇄`. **No orphan `<bdi>k</bdi>` anywhere.**
- Rulebook sentence `$80.00 לחודש על $10k` → `$10k` intact, LTR, `k` on the right.
- Audited every abbreviation site: account-size chips use `sizeLabel` (`$10,000` /
  `$100`, no `k`); the only displayed `$10k`/`10k` tokens are the target `≈` lines
  and the `$`-mode toggle label — all now glued.
- Service worker bumped **pt-dash-v13 → v14**.
