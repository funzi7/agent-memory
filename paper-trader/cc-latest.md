# paper-trader dashboard — cc-latest handoff

_Generated by a Claude Code session in the firewalled dev sandbox; verified with a
headless Chromium against the live published `portfolios.json`._

## What this change did
Render swing's structured pending orders properly on the dashboard + detail page.
Scope: `funzi7/agent-memory` only (`paper-trader/app.js`, `paper-trader/sw.js`).
SW cache bumped **pt-dash-v15 → v16**.

Swing pending orders now carry `{rank, price, qty}` per order. Previously the
dashboard re-derived affordability from `closes.json` (which doesn't price swing's
S&P-500 names), so they showed as plain buys, unpriced, alphabetical. Now swing
renders straight from its own structured fields via a new `pendingStructuredHTML`
(other strategies keep the closes-based preview).

### FIX 1 — price shown
Each swing pending row shows its price (e.g. `$262.15`), `$`/digits LTR-isolated per
the rulebook, ticker isolated.

### FIX 2 — ordered by rank
Swing pending is sorted by the integer `rank` ascending (1 first), not
alphabetically: EXPE(1) · MHK(2) · DHI(3) · SWK(4) · BKNG(5). Applied on both the
dashboard card and `strategy.html?id=swing`.

### FIX 3 — affordable (qty>0) vs unaffordable (qty=0)
- `qty>0` → real planned buy: `⏳ קנייה: {qty} יח׳ {ticker} @ ${price} · דירוג R · {reason}`.
- `qty=0` → `⏳ לא ניתנת לרכישה: {ticker} — יחידה ${price} · דירוג R · {reason}` (the
  existing amber/`.pend.skip` hourglass styling).
The redundant `#N` is stripped from the breakout reason (`פריצה #1 …` → `פריצה …`)
since the rank is shown as `דירוג R`. Hebrew breakout/RSI context kept.
