## This change: durable anti-clobber fix (paper-trader) + summary-strip redesign (agent-memory)

### PART A — paper-trader (PR, the durable fix): publish DATA only
The daily state-publish was overwriting this dashboard every run. Root cause:
`paper-trader/tools/publish_state.py` shipped a **bundled old inline `dashboard.html`**
to agent-memory on every daily run, clobbering the maintained app.js RTL-rulebook
dashboard that **lives here in agent-memory**.

Fixed in **paper-trader PR #5** (`claude/publish-no-frontend` → main, Codex review):
- `dashboard.html` removed from the publish set — the publish now pushes **DATA ONLY**:
  `portfolios.json`, `closes.json`, `trades.csv`. No `.html/.js/.css` is ever uploaded.
- The bundled `dashboard.html` was **deleted** from the paper-trader repo (it existed
  only to be published) so it can't be shipped again. The dashboard front-end
  (`dashboard.html`, `strategy.html`, `app.js`, `style.css`, `sw.js`, manifest, icons)
  is **owned by agent-memory**.
- selftest `test_publish_data_only` locks it in (no bundled front-end; every published
  dest path is `.json`/`.csv`). 436 checks green. CONTEXT.md updated.

**Result: `dashboard.html` is NO LONGER in paper-trader's publish set**, so the daily
run stops erasing this dashboard.

### PART B — agent-memory (this commit): summary-strip "סיכום" redesign
The previous overlap fix (min-width + `word-break:break-word`) distorted the strip
(names broke mid-word across lines, uneven block heights). Reverted to the original
clean layout and solved the name collision non-destructively:
- Restored `.strip` to its original look: `minmax(150px,1fr)` grid, `.v` at 1.15rem
  bold tabular — the % stats look exactly as before, aligned and symmetric.
- "הטובה ביותר היום" / "החלשה ביותר היום" strategy names now sit on their OWN single
  line under the label and **ellipsize** if too wide (`white-space:nowrap; overflow:
  hidden; text-overflow:ellipsis`), with the **full name in a `title` tooltip**. The
  two stat blocks stay aligned and never overlap, on phone and wide widths.
  (`.stat{min-width:0}` is the only structural addition — it lets the cell clip to an
  ellipsis instead of overflowing into its neighbour.)
- **The other 4 fixes from last change are intact** (app.js untouched this commit):
  Hebrew "דירוג N" trade rows (proper CSV parse, no stray quote), compact
  רענון+אפס-יעדים header, entry (כניסה $X) + current (נוכחי $Y) price per holding,
  and per-ticker rank derived from each holding's latest BUY reason.
- SW cache bumped **pt-dash-v9 → pt-dash-v10**.

Files (agent-memory): `dashboard.html`, `style.css`, `sw.js`, `cc-latest.md`.
