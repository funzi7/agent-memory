## Feature 1 — partial exit / scale-out
When an eligible strategy's KEPT position is up ≥ `trigger_gain` (0.15), the engine
queues a one-time partial SELL of `sell_fraction` (0.5, whole shares, ≥1 each side;
1-share skips) — reason "מימוש חלקי +N% — נמכרה מחצית, סטופ הועלה לכניסה". Fills T+1,
reduces cost basis proportionally, records realized P&L, sets
`Position.partial_exit_done`. Then the stop is at **breakeven**: a close ≤ entry exits
the remainder ("סטופ הועלה לנקודת הכניסה — מתחת לכניסה"). Full exits (stop-loss etc.)
take precedence.
- Applies to **swing, momentum_scan, leveraged_momentum, screener_track, guru_track**;
  **benchmark / rsi2 / trend200 / rotation excluded**. Config `partial_exit:` with a
  per-strategy override. Full-sell path byte-identical.

### Dashboard note (agent-memory side, NOT changed here)
Partial pending orders are `{side:"SELL", qty, partial:true, reason}`. The dashboard
currently renders a pending sell as "מכירה: TICKER"; if you want it to surface the
"מימוש חלקי" reason / the partial qty, that's a future dashboard tweak (app.js). No
dashboard change was made in this task (paper-trader only).

## Feature 2 — derived `combined` portfolio (→ 20 portfolios)
`combined_100` / `combined_10k`: a read-only normalized blend of ALL strategies'
current holdings, **equal 1/9 per strategy** scaled to the FIXED account size (not the
~$90k sum), within-slice by weight%, summed per ticker, a shared ticker **deduped into
one position tagged `sources: [...]`**; cash = remainder; equity normalizes to the
size. Built each run from the 9 real portfolios (READ-ONLY — never orders, never
mutates them) and published with the 18. `combined.weights` allows a future custom
allocation.
- The **data-driven dashboard already renders them automatically** (strategy id
  `combined`). Holdings show ticker / weight% / value like any portfolio; the
  `sources` field is available on multi-strategy tickers for an optional "held by N
  strategies" note (future dashboard tweak).
