## What this change did
Two engine fixes so the daily cycle can't appear frozen and so a single-asset
strategy that can't afford its instrument behaves honestly. Opened **PR #8** to
`paper-trader` main (Codex review). Code scoped to `funzi7/paper-trader` only
(this agent-memory cc-latest is the explicit handoff step).

PR: https://github.com/funzi7/paper-trader/pull/8

## The stuck symptoms
Published state stuck at `last_processed_date: 2026-06-24`; swing_100/swing_10k
initialized but EMPTY (never scanned); trend200_100 with a perpetually-pending QQQM
buy that can never fill at $100.

## FIX 1 — date gate + first scan (`main.py`)
- **The gate is correct, not a freeze.** `date_T = effective_latest_date(bars)` is
  the latest *completed* bar (today's forming bar dropped only before 16:30 ET;
  past bars kept). The gate `prev_processed == date_str` skips full processing only
  when that bar was already processed, so the system **advances on the next run
  whose data carries a completed bar newer than the published date** (cron `30 22 *
  * 1-5` = 18:30 ET, after close). If it looks stuck, the fetched data has no newer
  completed bar yet (provider lag); the next run with fresh data advances. No forced
  advance (that would be look-ahead).
- **First scan of never-scanned portfolios:** in the equal-date branch, every
  portfolio with **no `equity_history`** (absent from state OR present-but-empty,
  e.g. swing) gets a FIRST SCAN via `engine.run_day` for `date_T` — swing runs its
  S&P 500 4-signal scan and queues T+1 buys (or stays cash). Portfolios that already
  have history are NOT re-run (no double-processing, no look-ahead), kept
  byte-for-byte, date unchanged.

## FIX 2 — single-asset unaffordable → cash + note (`engine.run_day`)
A single-asset strategy whose only instrument can't be bought whole at the account
size (trend200 → QQQM ≈$304 at $100; QQQM is already the cheapest US-listed
Nasdaq-100 share class) no longer queues a perpetual pending. If the whole target is
one name, nothing is held, and a whole share doesn't fit the cash, the engine drops
the order and records `"{tk} ${px} גבוה מההקצאה ${cash} — התיק נשאר במזומן"` — holds
cash. trend200 still tracks QQQ; the $10k trend200 holds QQQ normally. Multi-asset
`small_account_fill` partial buys and affordable single-asset (benchmark→SPYM) are
untouched.
