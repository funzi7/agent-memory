## What this change did
Fixed swing's small-account pending so an **unaffordable name is never shown as a
BUY**, and gave swing pending orders **structured `rank`/`price`/`qty`** fields.
Opened **PR #9** to `paper-trader` main (Codex review). Code scoped to
`funzi7/paper-trader` only (this agent-memory cc-latest is the explicit handoff).

PR: https://github.com/funzi7/paper-trader/pull/9

## The bugs
1. swing_100 (cash $100) listed EXPE (~$264) as a BUY. Root cause: swing pending
   orders carried no affordability info, **and** `closes.json` was missing swing's
   S&P-500 candidate prices (EXPE/MHK/DHI/SWK) — the equal-date first-scan published
   the pending but never refreshed `closes.json`, so the dashboard couldn't price
   them and showed plain buys (BKNG, which *was* priced, correctly showed
   unaffordable).
2. swing pending = `{ticker, side, reason}` only — rank buried in the reason text,
   no price/qty for the dashboard.

## Fix (swing only; other strategies byte-identical)
- New no-op `Strategy.decorate_orders` hook; **Swing overrides** it to attach to
  each BUY: `rank` (int, from the `פריצה #k` reason), `price` (latest close
  estimate), and `qty` — computed via the **same `plan_buys` small-account fill**
  the engine uses at execution. A $100 account gets **`qty=0` on every unaffordable
  name** (never a phantom buy) and `qty>0` only on the affordable subset; swing_10k
  fills normally. Every other strategy's `decorate_orders` is the no-op, so their
  pending orders stay `{ticker, side, reason}` byte-for-byte.
- `main.py` equal-date first-scan now also `save_closes(...)` so swing's candidate
  tickers are PRICED in the published `closes.json` (display-only; never affects
  fills).

### Dashboard note (agent-memory side, NOT changed here)
The published pending orders now carry `rank`/`price`/`qty`. If the dashboard reads
`o.qty === 0` (or `price > available cash`) it can show "לא ניתנת לרכישה — יחידה $X"
directly without re-deriving affordability, and sort by the integer `rank`. The
`closes.json` refresh also means the existing closes-based affordability now works
for swing's S&P names. No dashboard change was made in this task (paper-trader only).
