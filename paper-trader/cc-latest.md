- **Combined whole-share IS in main** — `git show origin/main:papertrader/combined.py`:

## The fix for the STALE DATA (not the code)
A normal next-day run self-corrects both — but would first EXECUTE the stale ARDX
SELL+BUY as a one-time **wash trade**. So this PR **extends the idempotent
`--recompute-pending` maintenance path** to also:
- re-derive the `full_reset` rebalancers' pending (`twitter_track`, `guru_track`) as NET
  deltas → clears the BUY+SELL collision without executing it;
- rebuild the derived `combined_*` pair whole-share, in place.
Real portfolios' positions/cash/history stay byte-identical (only pending + derived
combined change); date is NOT advanced; no fills. Operator triggers it via the daily
workflow's **`recompute_pending: true`** dispatch (real prices in CI).
