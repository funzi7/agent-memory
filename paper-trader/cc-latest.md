# paper-trader — cc-latest handoff

_Generated by a Claude Code session in the firewalled dev sandbox (offline selftest
+ synthetic preview; market data hosts firewalled here)._

## What this change did
Added a safe `--recompute-pending` maintenance path so stale pending orders left by
old code can be re-derived with the CURRENT logic — without re-filling or advancing
the date. Opened **PR #10** to `paper-trader` main (Codex review). Code scoped to
`funzi7/paper-trader` only (this agent-memory cc-latest is the explicit handoff).

PR: https://github.com/funzi7/paper-trader/pull/10

## Why
The swing fill + structured-pending fix (PR #9) is merged, but the PUBLISHED state
still shows stale pending from the 2026-06-24 processing done with OLD code:
swing_100 lists EXPE (~$264) as a BUY at $100, orders lack price/qty/rank. Because
`last_processed_date == 2026-06-24`, the engine won't reprocess that day, so the
corrected logic never re-runs against it.

## How (minimal, safe, idempotent)
- `engine.regenerate_pending(...)` — `run_day` steps 3–5c factored out (decide →
  cheap-remap → trailing stop → single-asset cash-note → `decorate_orders`).
  **Read-only** on cash/positions/high-water/realized history; sets only
  `pending_orders`. `run_day` now calls it → day behaviour **byte-identical**.
- `main.py --recompute-pending` — re-derive ONLY the next-open pending for the last
  processed day `D` (`state.last_processed_date`, NOT the latest/forming bar, so the
  still-open day is never processed), then republish **pending only** (every other
  field kept byte-for-byte) + refresh `closes.json` (display-only). No look-ahead
  (data ≤ `D`'s close), no re-fill, no date advance, no `trades.csv` change.
- `daily.yml` — new `workflow_dispatch` input `recompute_pending` to run it in CI.

After it: swing_100's EXPE/MHK/DHI/SWK → `qty=0` + price (NOT a BUY), affordable
subset `qty>0`, every order carries int `rank`/`price`/`qty`.

## ⚠️ The real recompute must run in CI (NOT done in this PR)
Market data (Yahoo/yfinance) is **firewalled in the dev sandbox** (403), so the real
2026-06-24 prices can't be fetched here and the live `portfolios.json` was NOT
corrected in this change (fabricating prices would be wrong). **To apply the fix:**
after PR #10 merges, run the **daily** workflow via `workflow_dispatch` with
**`recompute_pending: true`** — it regenerates pending with real prices, commits
`state/` (date stays 2026-06-24, positions/cash untouched), and publishes here so
the dashboard shows swing's EXPE etc. correctly as unaffordable. Safe to run even
with the market open.

## Rules honored
Code scoped to `funzi7/paper-trader` only. Recompute PENDING only — never re-fills
or double-counts; no look-ahead; does not process the still-open day; existing
filled history byte-identical; fee math + T→T+1 unchanged.
