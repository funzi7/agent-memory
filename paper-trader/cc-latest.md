## FEATURE 1 — stop-skip: don't stop-out a name still in the strategy's own top_n
A position is **NOT** sold on its **STOP-LOSS** if that same ticker still ranks within
the strategy's own `top_n` entry set (it would be re-bought right back) → it is **HELD**
instead, saving the round-trip + its fees.
- **Stop-loss ONLY.** Every other exit still fires: regime-off, personal-momentum-
  negative, take-profit / partial, time-stop, move-over-SMA, monthly buffer-drop, and
  the leveraged **crash-stop**. Only the trailing/stop-loss sell is skipped, and only
  while the name is inside the SAME strategy's `top_n`.
- **The 4 ranked strategies only:** `momentum_scan`, `leveraged_momentum`, `swing`,
  `screener_track` (the last has no stop-loss → documented no-op). NOT benchmark / rsi2 /
  trend200 / rotation / guru_track / twitter_track / combined.
- **Mechanism:** shared `Strategy.top_entry_set(window)` (overridden by the 4 ranked
  strategies, respecting each one's OWN entry gate — e.g. momentum_scan excludes below-
  personal-exit-threshold names) + `stop_skip_enabled()`. The skip is injected at every
  stop-loss site BEFORE any refill, so allocation never exceeds `top_n` (engine
  `trail_pct` step; leveraged's 3 adaptive-trail sites; swing's internal `סטופ הפסד`).
- **Reversible:** `stop_skip_if_top_ranked.enabled` per strategy (default **true**).
  Hebrew run-note when it fires: `סטופ לא בוצע — עדיין בטופ {n} לכניסה, מוחזק`.

## FEATURE 2 — earnings-catalyst awareness (Modes A + D; B reserved OFF)
New `papertrader/earnings.py`: ONE batched **Finnhub earnings-calendar** call over held
+ entry-candidate tickers (`FINNHUB_API_KEY`), `numpy.busday_count` → trading days.
- **Mode A — don't ENTER right before earnings (default ON):** `run_catalyst` runs in
  `main.py` BEFORE `engine.run_day` and sets an entry **blackout** (report within
  `blackout_days`, default 3) on the ranked strategies; each skips a blackout name in its
  entry/top-up/refill loop so **the next-ranked eligible name fills the slot**. Held
  names are never blocked; guru_track/twitter_track (explicit picks) are never blocked.
  Hebrew note: `דילוג על כניסה {TK} — דוח רווחים בעוד {N} ימים`.
- **Mode D — WARN everywhere (default ON, display-only):** after the cycle,
  `flag_positions` (+ `flag_dicts` for the derived combined pair) stamps
  `earnings_in_days: N` on every held position + pending order within `warn_days`
  (default 7), across ALL portfolios. Report renders `⚠️ דוח רווחים בעוד {N} ימים` on
  holding + pending-buy lines. Never changes holdings.
- **Mode B — auto-exit before earnings: reserved OFF** (`mode_b_exit_before_earnings:
  false`), not implemented.
- **Non-blocking + graceful:** no key / HTTP / network / bad body / unknown date ⇒ empty
  map, never guessed, never crashes the cycle. `http_get` injectable (offline-tested);
  `--synthetic` fabricates dates; `FINNHUB_API_KEY` scrubbed from every logged string.
