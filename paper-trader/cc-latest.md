# paper-trader dashboard — cc-latest handoff

_Generated by a Claude Code session in the firewalled dev sandbox; widths,
overlaps and wrapping measured with a headless Chromium against the real published
data (18 portfolios incl. swing) at six viewport widths._

## What this change did
The overlap fix was correct but **over-shrank the cards**: `minmax(340px,1fr)`
inside a `max-width:1160px` container packed 3 narrow columns whose two inner
sub-cards (~165px each) wrapped almost every Hebrew word/number onto its own line —
tall thin strips. This change makes the cards **wide and readable**, using the
page's horizontal space, while staying a pure non-overlapping grid. Scope:
`funzi7/agent-memory` only (`paper-trader/style.css`, `paper-trader/sw.js`).

## What changed
- **Container uses page width:** `main` `max-width` **1160px → 1600px** (the cards,
  summary strip, chart and trades all get the extra room).
- **Wider columns:** `.cards` →
  `grid-template-columns: repeat(auto-fit, minmax(min(100%, 560px), 1fr)); gap:20px;
  align-items:start;`. The 560px minimum yields ~2 WIDE columns on desktop; the
  `min(100%, …)` guard collapses to ONE full-width column on phones so it never
  overflows. Still pure grid — no multi-column / masonry / float / absolute, so
  cards can never overlap.
- **Inner $100/$10k sub-cards consistently wide:** `.sizes` →
  `repeat(auto-fit, minmax(240px,1fr))` (was a fixed `1fr 1fr` + a 470px stack
  query). They stay side-by-side and wide inside the wide outer card and collapse
  to one full-width column only when the card itself is narrow (phones / a single
  full-width card) — matching the outer card's roomy feel, not thin strips.
- **No more word-by-word wrapping:** holdings tables now use `table-layout:auto`
  (was `fixed`, which equalised columns and starved the ticker/sub column). The
  ticker column now takes the room it needs so the holding line (טיקר · דירוג ·
  כניסה · נוכחי · רו״ה) reads on 1–2 natural lines; numeric columns stay tight;
  long `$` tokens wrap only as a last resort (`overflow-wrap:anywhere`).
- Service worker bumped **pt-dash-v12 → v13** so clients pick up the new stylesheet.

Card width is **≈774px at 1920px** — well above the old 340px. **0 overlaps at every
width**; **0** cards exceed the viewport; mobile (390px) is a single full-width
column with no horizontal overflow; holdings rows wrap to at most 2 clean lines
(never one token per line). Re-verified with a data-heavy stress card (8 holdings
injected into swing): identical — 0 overlaps, ≤2-line holding rows. 0 JS errors.

## Rules honored
Scope `funzi7/agent-memory` only. Pure CSS grid — no multi-column/masonry/absolute/
float; mobile collapses to one full-width column (no overflow); outer card and the
inner $100/$10k sub-cards share the same wide, uncramped treatment. Read-only page;
text follows `paper-trader/RTL-RULEBOOK.md` (layout-only change; `applyRtl()`
untouched).
