# paper-trader dashboard — cc-latest handoff

_Generated by a Claude Code session in the firewalled dev sandbox; overlap proven
with a headless Chromium at six viewport widths against the real published data._

## What this change did
Make the strategy-cards layout **provably overlap-proof at any width** and bump the
service-worker cache so clients actually pick up the corrected stylesheet. Scope:
`funzi7/agent-memory` only (`paper-trader/style.css`, `paper-trader/sw.js`).

## Diagnosis — the real cause
- I checked for the usual overlap culprits in `style.css`/`app.js`: **none present**
  — no CSS multi-column (`column-count`/`columns`), no JS masonry, no `float`, no
  `position:absolute`, no negative margins, no fixed card heights. `.cards` was
  already a plain `display:grid`.
- A headless 6-width overlap test (pairwise `getBoundingClientRect` intersection)
  on the **currently published files** (18 portfolios incl. swing) returned **0
  overlaps at 2560/1920/1440/1024/768/390px**. So the live grid does not overlap in
  a browser.
- Therefore the overlap users still saw is a **stale service-worker shell cache**:
  `sw.js` serves shell assets (`style.css`, `app.js`) **cache-first**, so an
  installed PWA kept serving an OLDER stylesheet (a pre-grid version that did
  overlap) until the cache was evicted. The fix must both (a) make the CSS
  bulletproof and (b) bump the SW version to force the corrected CSS to clients.

## What replaced it (pure grid, cannot overlap)
`.cards` is now:
```css
.cards{ display:grid; grid-template-columns:repeat(auto-fit,minmax(340px,1fr));
  gap:20px; align-items:start; }
```
Each card owns its own grid CELL, so a data-heavy card only makes its column
TALLER — it can never slide behind a neighbour. Per-card defensive resets clear any
inherited bad layout: `min-width:0; width:auto; height:auto; position:static;
float:none; margin:0;`. Inner content is contained so it never widens the cell:
`overflow-wrap:anywhere` on text rows (`.desc`/`.cashline`/`.size`/cells),
`min-width:0` on the inner `.sizes` columns, and `table-layout:fixed` on the
in-card holdings tables (the wider recent-trades table stays auto and scrolls
inside its own `.tablewrap`, never the page/grid). No multi-column, masonry,
absolute or float anywhere for cards. Service worker bumped **pt-dash-v11 → v12**
(activate purges every non-v12 cache, `skipWaiting` + `clients.claim`), so clients
load the corrected stylesheet.
