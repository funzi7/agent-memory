## How to do the REAL run + push yourself (PC where Yahoo is reachable)
In the `quality-screener` folder, with your PAT, **Windows PowerShell**:

```powershell
$env:CROSS_REPO_PAT = "YOUR_PAT_HERE"
python main.py            # real run: fetches Yahoo, screens, writes screener_picks.yaml
python tools\push_picks.py  # GET sha (if exists) -> PUT to funzi7/paper-trader default branch
```

macOS/Linux (bash):
```bash
CROSS_REPO_PAT="YOUR_PAT_HERE" python main.py && python tools/push_picks.py
```

Notes:
- `main.py` (non-dry run) writes `screener_picks.yaml` locally; `tools/push_picks.py`
  reads `CROSS_REPO_PAT` + that file and PUTs it to `funzi7/paper-trader` (default
  branch) with message `screener picks: {date}`. The PAT is scrubbed from any
  logged URL/error; a push failure logs a warning and never crashes the run.
- Telegram secrets are optional — if unset, the send is a no-op (no crash).
- The push lands `screener_picks.yaml` in paper-trader, which `screener_track`
  trades monthly (no regime/trailing — measuring the raw screen).
